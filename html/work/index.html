잡지 속 뷰티샷을 완성하는 포토샵 고급 기술

  • 뷰티 아이템 촬영의 기초와 핵심
    • 정보창 : rgb와 cmyk의 데이터 값을 확인할 수 있습니다.
    • 스포이드를 통해 잉크 총 사용량을 확인할 수 있습니다.(0에 가까우면 하얀색, 350에 가까워지면 검은색 7%미만은 하얀색 출력 X. 299%이상은 블랙만 출력
  • 럭셔리 하고 몽환적인 분위기의 배경 만들기
    • 배경톤은 보통 contrast를 조절한다.
    • 레이어창의 반달모양 아이콘을 클릭-곡선 : 후반 작업에서 수정 용이 손가락 아이콘을 클릭하면 스포이드가 나온다.
    • 밝기 위치를 잡아줌 오른쪽 화이트 중앙 회색 왼쪽 블렉 원하는 부분을 잡고 끌어 올리거나 내리면 톤 밝기 조정 가능 위로 올리면 밝아지고 아래로 내리면 어두워진다.
  • 흐르는 방울 및 리퀴드 표현 및 합성 방법
    • 필터-픽섹유동화(숫자가 작을수록 뒤틀림이 작아짐)
    • 방울라인과 물방울의 끝 라인이 맞도록 마스크를 이용하여 자연스럽게 붙여줍니다.
    • 가우시안 흐림효과(포커스 아웃) path를 이용
  • 어려워도 피할 수 없는 리퀴드 촬영과 리터칭 기술
    • 불투명도를 조절해서 경계선 위치를 잡아준다.(편집 자유변형),합성작업은 주로마스크를 이용한다. 옵션 백 마스크선택수정 매끄럽게
    • 흐르는 방울 합성 : 마스크를 이용하여 필요없는 부분을 정리해준다 (어려운 작업은 패스로 선택하여 정리) -> 마지막은 브러쉬를 이용해 깔끔하게 정리해준다.
    • 촬영할때 더 디테일하게 촬영하여 합성소스가 많을수록 디테일해 진다.
  • 수분감 팡팡 터지는 이미지를 위한 물 모양 합성 작업
    • 스튜디오에서 조명을 이용하여 촬영 듀레이션이 빠른 조명을 사용 4000/1
    • 합성할 이미지를 불투명도를 줄여 마스크를 이용해 합성하고 브러쉬의 정도와 불투명도를 이용해 정리해준다
    • 물모양 합성시 뒤에 깔끔하지 못한 물방울들은 지워준다.

생활코딩

HTML

HTML(HyperText Markup Language)

  • HT = HyperText : 하이퍼텍스트를 가장 중요한 특징으로 하는
  • M = Markup : 마크업이라는 형식을 가진
  • L = Language : 컴퓨터 프로그래밍 언어 (약속)
  • HTML이란? : 하이퍼텍스트를 가장 중요한 특징으로 하는 마크업이라는 형식을 가진 컴퓨터 프로그래밍 언어를 말합니다.

태그(Tag)

HTML에서 사용되는 기본적인 문법

  • 태그는 열린태그(<)와 닫는 태그(/>)가 있습니다.
  • 닫는 태그에는 "/"있어야 합니다.

<h1>~<h6>

(heading의 약자 = 제목), 제목 영역을 설정하는 태그 입니다.

  • <h1>은 가장 중요한 제목을 정의합니다. <h6>는 가장 중요하지 않은 머리글을 정의합니다.

<strong>

강조할때 사용되는 태그 입니다.

<a>

하이퍼 링크를 정의합니다.

  • href : 링크로 이동하는 경로를 설정합니다.
  • target : 브라우저의 타겟을 설정합니다.
    • <a target="_blank"></a> : 링크 클릭시 새로운 브라우저 창으로 설정합니다.

<ul>

(unorderd list) : 순서가 없는 항목을 정렬합니다.

<li>

(list) : 각각의 항목을 설정합니다.

<ol>

(orderd list) : 순서가 있는 항목을 정렬합니다.

<html>

HTML 문서의 가장 기본이 되는 태그 입니다.

<head>

본문이 아닌 태그들을 정렬합니다.

<title>

탭의 제목을 설정합니다.

<meta>

웹페이지에 표현되진 않지만, 웹 페이지를 설명하기 위한 부가 정보들을 담당합니다.(닫는 태그가 없는 태그)

<!-- html5 -->
<meta charset="UTF-8">
  • charset="속성값" : 브라우저에 디코딩되는 문자형식을 지정해 줍니다. (속성값 utf-8은 한글폰트를 정상 출력시켜줍니다.)
  • name="description" content="내용" : 검색 결과에 대한 요약에 사용됩니다.
  • name="keywords" content="내용" : 문서를 설명하는 중요 단어(키워드)들 입니다.
  • name="author" content="내용" : 저자를 의미합니다.
  • http-equiv="refresh" content="숫자" : 지정한 시간(초) 마다 웹페이지를 리프레쉬 시켜줍니다.(자주 사용하진 않음)

<body>

본문의 내용들을 정렬합니다.

<button>

버튼을 만들어주는 역활을 하는 태그 입니다.

<img>

이미지를 표시하는 태그 입니다.

<script>

Java Script(미국 넷스케이프 커뮤니케이션스가 개발한 스크립트 언어)를 쓸때 사용하는 태그 입니다.

<div>

웹 페이지를 꾸며줄때 그룹으로 나누어주는 태그 입니다.

주요태그

<p>

단락(paragraph)을 구분할때 사용되는 태그 입니다.

  • 브라우저에게 어디서부터 어디까지가 단락이라는걸 알려줄때 단란의 시작과 끝에 <p>태그로 감싸줍니다.
  • <p>태그에는 기본적인 간격이 있습니다. 이는 css를 통해 조정이 가능합니다.

<br>

줄바꿈 태그 입니다.(닫는 태그가 없는 태그)

  • 내용이 없는 태그기 때문에 닫는태그가 없는 태그입니다.
  • <br>태그는 연속해서 사용이 가능합니다.

<img>

웹 페이지에 이미지를 포함시켜 줍니다.(닫는 태그가 없는 태그)

  • 원하는 이미지를 디렉토리 폴더에 넣고 src=""속성을 이용해 파일의 이름을 적어 불러옵니다.
  • width값이나 hight값을 이용해 사진의 크기를 조정할 수 있습니다.
  • alt="" 속성 (alternative text = 대체)
    • 이미지가 깨져서 볼 수 없을 경우에 대한으로 사용 가능합니다.
    • 특히 시각장애인들의 경우에는 img태그만을 이용한 이미지를 볼 수 없기 때문에 그자리에 어떤 정보가 위치하는지 적어주어야 합니다.
  • title="" 속성
    • 마우스를 올려놨을때 도움말이 나와 이미지의 제목을 보여줍니다.
<img src="img.jpg" height="300" alt="산 이미지" title="산 이미지">

<table>

표에 해당되는 태그 입니다.

  • border값을 주면 표의 테두리를 나타낼 수 있습니다.

<tr>

행태그(table row)

<form>

사용자가 입력한 정보를 서버로 전송할때 사용하는 태그입니다.

  • action="URI" : 사용자가 입력한 정보를 지정한 URI로 전송해 줍니다.
  • method="HTTP메소드" : 데이터를 어떤 방법으로 전송할 것인가 내용을 처리하는 방법을 지정합니다.
    • method="get" : url을 통해 데이터를 전송합니다.
    • method="post" : action 에서 지정한 URI을 다른 방식으로 데이터를 숨겨서 전송해 줍니다.
  • enctype=""속성 : 폼을 전송할때 사용할 인코딩 방법을 지정합니다.
    • enctype="multipart/form-data" : 어떠한 문자 인코딩도 하지 않습니다. 주로 파일 업로드를 하는 경우 사용합니다.

<input>

사용자로 부터 정보를 입력 받을 수 있습니다.(닫는 태그가 없는 태그)

  • type=""(필수 속성) : 사용자로부터 정보를 받을 수 있는 박스가 만들어집니다.
    • type="text" : 사용자로부터 문자정보를 받을 수 있는 박스가 만들어집니다.
    • type="password" : 사용자로부터 비밀번호를 받을 수 있는 박스가 만들어집니다.
    • type="submit" : 정보를 제출할 수 있는 버튼을 만들어 줍니다.
    • type="radio" : 라디오버튼을 생성 합니다. name="" 값을 같게주면 한개만 선택이 가능합니다.(복수 선택 불가능)
    • type="checkbox" : 체크박스를 생성합니다. name="" 값을 같게주면 여러개 체크가 가능합니다.(복수 선택이 가능)
    • type="button" : 버튼을 생성합니다. (javascript등을 함께 사용할때 활용도가 생기는 버튼)
    • type="reset" : 재설정 버틀을 생성합니다. 이 버튼은 박스안의 입력된 정보를 초기화 시켜줍니다.
    • type="hidden" : 감춰진 필드를 말합니다. 눈에 보이지는 않지만 서버로 어떤 정보를 전송할때 사용합니다.
    • type="file" : 파일을 송신할 수 있게 해줍니다.
  • name="이름"(필수 속성) : type속성에 입력된 값에 이름을 부여해 줍니다.
    • name="id" : 입력된 텍스트가 아이디로 식별됩니다.
    • name="password" : 입력된 텍스트가 비밀번호로 식별됩니다.
    • name="address" : 입력된 텍스트가 주소로 식별됩니다.
  • value="값" : 사용자가 페이지를 열었을때 기본값으로 박스안에 보여집니다.
  • checked : type="radio"와type="checkbox"를 만들때 기본적으로 선택됬으면 하는 부분에 사용하면 초기화면에 선택되어 보여집니다.

ex)

아이디 :

<form action="http://localhost/login.php">
    <p>아이디 : <input type="text" name="id" value="default value"></p>

    <input type="submit">
</form>

<select>

선택목록을 생성합니다.

  • name="이름" : select속성에 입력된 값에 이름을 부여해 줍니다.
  • multiple=""속성 : ctrl을 누르면 다중으로 선택이 가능합니다.

<option>

각 항목을 정의합니다.

  • value="값" : 해당 항목이 선택된 때에 서버에 전송되는 값을 지정합니다.

ex)

색상 (다중선택)

<form action="http://localhost/color.php">
    <h3>색상 (다중선택)</h3>
    <select name="color" multiple>
        <option value="red">붉은색</option>
        <option value="black">검은색</option>
        <option value="blue">파란색</option>
    </select>
    <input type="submit">
</form>

  • rows="행수"(필수 속성) 줄을 지정할 수 있습니다.

ex)

textarea :

퇴출된 태그

<font>

문서 내의 text를 시각적으로 꾸며주는 역할을 담당합니다.

  • size="크기" : 기본크기는 3이며 1~7까지의 값을 표현해낼 수 있습니다.
  • color="색" : 지정한 색으로 바꾸어 표현됩니다.
  • 디자인의 역할만 할 뿐 어떠한 정보로서의 의미도 갖지 않으며,중복 사용으로 인한 파일 용량 증가, 가독성 저하, 관리 측면에서의 불편함 등의 이유로현재는 퇴출된 태그이며 그 역할은 CSS가 담당합니다.

DOCTYPE선언

DTD(Doucument Type Declaration)

자신이 작성한 HTML코드가 어떤 방식의 HTML코드로 작성됬는지를 알려주는 문자 입니다.

  • D = Doucument : 문서
  • T = Type : 형식
  • D = Declaration : 선언,공표하다
  • DTD이란? : 브라우저에게 어떠한 표준을 따르고 있는 태그들인지 알려주는것을 말합니다.
<!-- html5 -->

숫자

자바스크립트에서 큰따옴표나 작은따옴표가 붙지 않은 숫자는 숫자로 인식합니다.

  • alert(); 괄호안에 어떠한 값을 주면 경고창이 뜨면서 괄호안에 넣은 값이 경고창에 표시됩니다.
  • 정수 = 자연수 ex) -3 -2 -1 0 1 2 3
  • 실수 = 소수점이 존재하는 숫자 ex) -0.2 -0.1 0 0.1 0.2 0.3
  • + : 값을 더하고 싶을때 
    • alert(1+1); //결과 : 2

    alert(1.2 + 1.3); //결과 : 2.5
  • * : 값을 곱하고 싶을때
    • alert(2 * 5); //결과 : 10
  • / : 값을 나누고 싶을때 
    • alert(6 / 2) //결과 : 3
  • Math : 여러가지 연산을 할때 사용합니다.
  • pow : 제곱
    • Math.pow(3,2); //(= 3의 2승) 결과 : 9
  • round : 반올림
    • Math.round(10.6); //(= 10.6을 반올림) 결과 : 11
  • ceil : 올림
    • Math.ceil(10.2); //(= 10.2를 올림) 결과 : 11
  • floor : 내림
    • Math.floor(10.6); //(= 10.6을 내림) 결과 : 10
  • sqrt : 제곱근
    • Math.sqrt(9); //(= 3의 제곱근) 결과 : 3
  • random : = 0부터 1.0 사이의 랜덤한 숫자
    • Math.random();

문자

""(큰 따옴표) 혹은 ''(작은 따옴표) 중의 하나로 감싸야 합니다.

  • 숫자를 따옴표로 감싸면 문자가 됩니다.
    • alert(typeof "1") //결과 : string(=문자열)
  • 따옴표 없는 숫자는 number로 출력됩니다.
    • alert(typeof 1) //결과 : number
  • \ : 문자열 안에 작은 따옴표나 큰따옴표를 넣을때 사용합니다.(escape)
    • alert('egoing\'s javascript') //결과 : egoing's javascript
  • \n: 줄바꿈을 의미하는 특수한 문자입니다.
    • alert("안녕하세요.\n생활코딩의 세계에 오신 것을 환영합니다");
  • + : 문자와 문자를 더할 때
    • alert("coding"+"everybody"); //결과 : codingeverybody
alert("coding" + "everybody"); //결과 : coding everybody
  • length : 문자의 길이를 구할 때는 문자 뒤에 붙입니다.
    • alert("coding everybody".length) //결과 : 16
  • indexOf : 입력한 값이 문자열에서 몇번째에 위치하는지 알려줍니다.
    • "code".indexOf("c") //결과 : 0
"code".indexOf("o") //결과 : 1
"code".indexOf("d") //결과 : 2
"code".indexOf("e") //결과 : 3

  • 변수의 값이 꼭 숫자만 올 수 있는것은 아닙니다.
    • var first = "coding";
alert(first+" everybody"); //결과 : coding everybody

var a = 'coding', b = 'everybody';

alert((100+10)/10);
alert(((100+10)/10)-10);

    a = 100;

// (주석)

주석(comment)으로 코드에 부가적인 설명을 쓰거나 사용하지 않는 코드를 비활성화시키기 위해서 사용합니다. ( // 뒤에 따라오는 내용은 해석되지 않습니다.)

  • // : 사용된 줄에만 주석기능이 처리됩니다.
  • /* */ : 여러줄을 주석처리 해줄때 사용합니다.

; (줄바꿈과 여백)

세미콜론은 하나의 구문이 끝났음을 명시적으로 나타내는 기호입니다. (한줄에 여러구문을 사용하고 싶을 때 유용합니다.)

  • JavaScript에서는 세미콜론을 생략할 수 있는데, 이 경우 줄바꿈을 명령의 끝으로 간주하게 됩니다.

비교


= (비교)

값에 대해서 어떤 작업을 컴퓨터에게 지시하기 위한 기호 입니다.

a=1
  • '='(=대입 연산자)는 1(=값)을 a(=변수)에 대입하는 '대입 연산자'입니다.
  • '값'에는 변수에 대응되는 수만 올 수 있습니다.

비교 연산자

프로그래밍에서 비교란 주어진 값들이 같은지, 다른지, 큰지, 작은지를 구분하는 것을 의미합니다.

  • 비교 연산자의 결과는 true나 false 중의 하나 입니다.
  • true,false 블린(boolean)이라고 불리는 데이터 형식입니다.
  • true : 참
  • false : 거짓

== (동등 연산자)

좌항과 우항을 비교해서 서로 값이 같다면 true 다르다면 false가 됩니다.

alert(1==2)             //false
alert(1==1)             //true
alert("one"=="two")     //false 
alert("one"=="one")     //true
  • '='가 하나인 것 : 우항의 값을 좌항의 변수에 대입할 때 사용하는 것(대입 연산자)
  • '='가 두개인 것 : 좌항과 우항의 값이 같으면 true(참) 다르면 false(거짓)이라고 나타내주는 것(동등 연산자)

=== (일치 연산자)

좌항과 우항이 '정확'하게 같을 때 true 다르면 false가 됩니다.

alert(1=='1');          //true
alert(1==='1');         //false
  • '==='는 숫자 1과 문자 1을 다르게 인식합니다.
  • 서로 같은 수를 표현하고 있더라도 데이터 형이 같은 경우에만 같다고 판단하기 때문입니다.
  • == 연산자 대신 === 연산자를 쓰는 것을 강력하게 권합니다.

동등 연사자와 일치 연산자의 비교

alert(null == undefined);       //true
alert(null === undefined);      //false
  • null = 값이 없다.
  • undefined = 값이 정의되지 않았다.(값이 없는 상태)
  • == (동등 연산자)는 입력한 데이터가 달라도 의미가 같으면 같다고(true) 해석합니다.
  • === (일치 연산자)는 입력한 데이터의 의미가 같아도 데이터가 다르면 다르다고(false) 해석합니다.
alert(true == 1);               //true
alert(true === 1);              //false
alert(true == '1');             //true
alert(true === '1');            //false
  • == (동등 연산자)는 숫자1을 true로 간주합니다.
  • == (동등 연산자)는 숫자1이 아닌 숫자를 false로 간주합니다.
alert(0 === -0);                //true
alert(NaN === NaN);             //false
  • 0은 양수,음수 모두 정 가운데 숫자이기 때문에 데이터가 같습니다.
  • 0은 양수,음수 모두 정 가운데 숫자이기 때문에 데이터가 같습니다.
  • NaN은 0/0과 같은 연산의 결과로 만들어지는 특수한 데이터 형인데 숫자가 아니라는 뜻 입니다.

!=

부정을 의미합니다.(같지 않다)

alert(1!=2);            //true
alert(1!=1);            //false
alert("one"!="two");    //true
alert("one"!="one");    //false
  • '같다'(=)의 부정은 '같지 않다'(!=)입니다.

!==

부정을 의미합니다.(정확하게 같지 않다)

  • '정확하게 같다'(==)의 부정은 '정확하게 같지 않다'(!==)입니다.

>

좌항이 우항보다 크다면 참(true), 그렇지 않다면 거짓(false)임을 알려주는 연산자 입니다.

alert(10>20);   //false
alert(10>1);    //true
alert(10>10);   //false

>=

좌항이 우항보다 크거나 같다.

alert(10>=20);    //false
alert(10>=1);     //true
alert(10>=10);    //true

조건문


Boolean

숫자와 문자처럼 언어에서 제공하는 데이터 형입니다.

  • 불린으로 올 수 있는 값은 true와 false 밖에 없습니다.
  • 조건문에서 핵심적인 역할을 담당 합니다.

조건문

주어진 조건에 따라서 에플리케이션을 다르게 동작하도록 하는 것이다.

  • // : 사용된 줄에만 주석기능이 처리됩니다.
  • /* */ : 여러줄을 주석처리 해줄때 사용합니다.

if

조건문은 if로 시작 합니다.

  • if 뒤의 괄호에 조건이 오고, 조건이 될 수 있는 값는 Boolean 입니다.
  • Boolean의 값이 true라면 조건이 담겨진 괄호 다음의 중괄호 구문이 실행 됩니다.
  • if 조건문의 조건이 참이면 중괄호의 시작({}부터 중괄호의 끝{})까지의 구간이 실행됩니다.
if(true){
    alert('result : true');   
}                               //결과 : 'result : true'
  • if 뒤에 true가 왔기 때문에 결과는 true 입니다.
if(false){
    alert('result : true');   
}                               //결과 : 아무것도 출력되지 않는다.
  • if 뒤에 false가 왔기 때문입니다.

else

주어진 조건이 거짓일 때 실행할 구간을 정의하는 것 입니다.

if(true){
    alert(1);
} else {
    alert(2);
}                               //결과 : 1

if(false){
    alert(1);
} else {
    alert(2);
}                              //결과 : 2
  • if문의 조건이 true라면 if의 중괄호 구간이 실행되고, false라면 else 이후의 중괄호 구간이 실행 됩니다.

else if

좀 더 다양한 케이스의 조건을 검사할 수 있는 기회를 제공 합니다.

  • if나 else와는 다르게 여러개가 올 수 있습니다.
  • else if의 모든 조건이 false라면 else가 실행됩니다.
  • else는 생략 가능합니다.
if(false){
    alert(1);
} else if(true){
    alert(2);
} else if(true){
    alert(3);
} else {
    alert(4);
}                             //결과 : 2

if(false){
    alert(1);
} else if(false){
    alert(2);
} else if(true){
    alert(3);
} else {
    alert(4);
}                             //결과 : 3

if(false){
    alert(1);
} else if(false){
    alert(2);
} else if(false){
    alert(3);
} else {
    alert(4);
}                             //결과 : 4

변수와 비교연산자


prompt

사용자가 입력한 값을 가져와서 id 변수의 값으로 대입합니다.(API 또는 함수라고 부릅니다.)

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8"/>
</head>
<body>
    <script>
        var id = prompt('아이디를 입력해주세요.')
        if(id=='egoing'){
            alert('아이디가 일치 합니다.')
        } else {
            alert('아이디가 일치하지 않습니다.')
        }
    </script>
</body>
</html>
  • 사용자가 입력한 값이 egoing이라면 '아이디가 일치 합니다'를 출력하고 그렇지 않다면 '아이디가 일치하지 않습니다'를 출력합니다.

조건문의 중첩

조건문은 조건문 안에 중첩해서 사용될 수 있다.(if문 안에 다시 if문)

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8"/>
</head>
<body>
    <script>
        id = prompt('아이디를 입력해주세요.');
        if(id=='egoing'){
            password = prompt('비밀번호를 입력해주세요.');
            if(password==='111111'){
                alert('인증 했습니다.');
            } else {
                alert('인증에 실패 했습니다.');
            }
        } else {
            alert('인증에 실패 했습니다.');
        }
    </script>
</body>
</html>
  • 사용자가 입력한 값이 111111이라면 '인증 했습니다'를 출력하고 그렇지 않다면 '인증에 실패 했습니다.'를 출력합니다.
  • 사용자가 입력한 값과 아이디의 값이 일치하는지를 확인한 후에 일치한다면 비밀번호가 일치하는지 확인 합니다.

논리 연산자

조건문을 좀 더 간결하고 다양한 방법으로 구사할 수 있도록 도와줍니다.


&&

좌항과 우항이 모두 참(true)일 경우에만 참이됩니다.(and 연산자)

if(true && true){
    alert(1);
}
if(true && false){
    alert(2);
}
if(false && true){
    alert(3);
}
if(false && false){
    alert(4);
}                             //결과 : 1

||

'||'의 좌우항 중에 하나라도 true라면 true가 됩니다.(or 연산자)

if(true || true){
    alert(1);
}
if(true || false){
    alert(2);
}
if(false || true){
    alert(3);
}
if(false || false){
    alert(4);
}                             //결과 : 1,2,3
  • '||'는 좌항과 우항이 모두 false이기 때문에 false가 됩니다.

!

부정의 의미로, Boolean의 값을 역전시킵니다.(not 연산자)

if(!true && !true){
    alert(1);
}
if(!false && !true){
    alert(2);
}
if(!true && !false){
    alert(3);
}
if(!false && !false){
    alert(4);
}                        //결과 : 4
  • true를 false로 false를 true로 만듭니다.

false로 간주되는 데이터 형

  • 빈문자열('')
  • undefined
  • null
  • NaN
  • 0

반복문

컴퓨터에게 반복적인 작업을 지시하는 방법입니다.


while

뒤에 따라오는 괄호 안의 조건이 참(true)면 중괄호 안의 코드 구간을 반복적으로 실행한다.

while (조건){
    반복해서 실행할 코드
}
  • 조건이 false면 반복문이 실행되지 않습니다.
  • true와 false는 종료조건이 되는데, 이 값을 변경하는 것을 통해서 반복문을 종료시킬 수 있습니다.
  • 반복문에서 종료조건을 잘못 지정하면 무한반복이 되거나, 반복문이 실행되지 않습니다.
var i = 0;
while(i < 10){
    document.write('coding everybody <br>');
    i =i++
}
  • 종료조건으로 i의 값이 10보다 작다면 true, 같거나 크다면 false가 됩니다.
  • 반복이 실행될 때마다 coding everybody <br>이 출력됩니다.
  • i의 값이 1씩 증가합니다.

for

while대신 사용할 수 있습니다.

for(초기화; 반복조건; 반복이 될 때마다 실행되는 코드){
    반복해서 실행될 코드
}

for(var i = 0; i < 10; i++){
    document.write('coding everybody'+i+'<br>');
}
  • i++ : 기존의 값을 사용하고 i를 증가시킵니다.
  • ++i : 사용하는 동시에 i값을 증가시킵니다.
  • +i+ : 숫자를 순서대로 붙이고 싶을때 사용합니다.

반복문이 없을때

  • 간단한 작업은 복사&붙여넣기를 이용해서도 할만합니다. 하지만 좀 더 큰 규모의 데이터를 다뤄야 한다면 반복문의 효용이 부각되기 시작합니다.
  • 반복문 없이 뒤에 숫자를 1부터 10까지 붙이고 싶다면 복사&붙여넣기도 불가능한 일입니다.
var i = 0;
while(i < 10){
    document.write('coding everybody '+i+'<br> ');
    i++;
}
  • 그러나 반복문에서는 뒤에 숫자를 1부터 10까지 붙이고 싶다면 이렇게 표현할 수 있습니다.
var i = 0;
while(i < 10){
    document.write('coding everybody '+(i*2)+'<br>');
    i++;
}
  • +(i*2) : 뒤에 붙는 숫자를 2의 배수하고 싶을 경우 사용됩니다.

반복문의 제어

반복작업을 중간에 중단시키고 싶을때 사용됩니다.


break

반복문을 즉시 종료시킵니다.

for(var i = 0; i < 10; i++){
    if(i === 5) {
        break;
    }
    document.write('coding everybody'+i+'<br>');
}
/*결과
coding everybody 0
coding everybody 1
coding everybody 2
coding everybody 3
coding everybody 4*/
  • 2행의 if(i === 5) 에 의해서 i의 값이 5일 때 break 문이 실행되면서 반복문이 완전히 종료된 것입니다.
  • 반복문 안에 조건문은 들어갈 수 있습니다.

continue

실행을 즉시 중단 하면서 반복은 지속돼게 해줍니다.

for(var i = 0; i < 10; i++){
    if(i === 5) {
        continue;
    }
    document.write('coding everybody'+i+'<br>');
}
/*결과
coding everybody 0
coding everybody 1
coding everybody 2
coding everybody 3
coding everybody 4
coding everybody 6
coding everybody 7
coding everybody 8
coding everybody 9*/
  • i의 값이 5가 되었을 때 실행이 중단 됐기 때문에 continue 이후의 구문이 실행되지 않습니다.

반복문의 중첩

반복문 안에는 다시 반복문이 나타날 수 있습니다.

// 0부터 9까지 변수 i에 순차적으로 값을 할당        
for(var i = 0; i < 10; i++){
    // 0부터 9까지의 변수를 j의 값에 순차적으로 할당
    for(var j = 0; j < 10; j++){    
        // i와 j의 값을 더한 후에 출력
        // String은 숫자인 i와 j의 데이터 타입을 문자로 형태를 변환하는 명령이다. 
        // String()을 제거하고 실행해보면 의미가 좀 더 분명하게 드러날 것이다.
        document.write(String(i)+String(j)+'<br>');
    }
}

함수


function(함수)

하나의 로직을 재실행 할 수 있도록 하는 것으로 코드의 재사용성을 높여줍니다.

function 함수명( [인자...[,인자]] ){
   코드
   return 반환값

  • function 뒤에 함수의 이름이 오고, 소괄호가 따라옵니다. 소괄호에 인자라는 값이 차례로 들어오는데 이 값은 함수를 호출할 때 함수의 로직으로 전달될 변수입니다.
  • 인자는 생략 할 수 있습니다.
  • 함수를 호출 했을 때 실행하게 될 부분이 중괄호 안쪽에 옵니다.
function numbering(){

        i += 1;
    }

numbering();                    //결과 :  0123456789

함수의 장점

  • 재사용성 : 동일한 코드를 여러곳에서 재사용 할수 있는 형태로 만듭니다.
  • 유지보수의 용이 : 함수를 사용하는 여러곳에서 한 코드 만으로 함수의 내용을 수정 , 개선, 변경이 가능합니다.
  • 가독성: 코드를 읽기 쉽습니다.

함수의 출력


return

return 뒤에 따라오는 값을 함수의 결과로 반환과 동시에 함수를 종료시킵니다.

function get_member1(){
    return 'egoing';
}
 
function get_member2(){
    return 'k8805';
}
 
alert(get_member1());
alert(get_member2());    //결과 : egoing, k8805
  • get_member1와 get_member2를 출력(alert)한 결과가 각각 egoing과 k8805인 이유는 함수 내에서 문자열 egoing과 k8805을 return을 하기 때문입니다.
function get_member(){
    return 'egoing';
    return 'k8805';
    return 'sorialgi';
}
alert(get_member());    //결과 : egoing
  • return 'egoing'을 실행한 후에 함수가 종료되었기 때문에 return 'egoing' 이하는 어떠한 경우도 실행되지 않습니다.

함수의 입력


인자(argument)

함수로 유입되는 입력 값을 의미하는데, 어떤 값을 인자로 전달하느냐에 따라서 함수가 반환하는 값이나 메소드의 동작방법을 다르게 할 수 있습니다.

function get_argument(arg){
    return arg;
}
 
alert(get_argument(1));
alert(get_argument(2));    //결과 : egoing 과 k8805
  • get_argument(1)의 괄호안 : 인자(argument)
  • 인자가 들어가는(arg) : 매개변수(parameter)
  • (arg) 구문에 의해서 변수 arg의 값으로 숫자 1이 함수 안으로 전달됩니다.
function get_arguments(arg1, arg2){
    return arg1 + arg2
}
 
alert(get_arguments(10, 20));
alert(get_arguments(20, 30));    //결과 : 30, 50
  • 함수를 호출 할 때 전달한 인자 10과 20은 함수의 arg1, arg2에 차례로 할당됩니다.

  • 일회성으로 사용됩니다.

데이터의 추가

배열은 복수의 데이터를 효율적으로 관리, 전달하기 위한 목적으로 고안된 데이터 타입입니다. 따라서 데이터의 추가/수정/삭제와 같은 일을 편리하게 할 수 있도록 돕는 다양한 기능을 가지고 있습니다.


length(크기)

배열에 담긴 값의 숫자를 알려줍니다.

var arr = [1, 2, 3, 4, 5];
alert(arr.length);         //결과 : 5

push(추가)

배열의 끝에 원소를 추가하는 방법입니다.

var li = ['a', 'b', 'c', 'd', 'e'];
li.push('f');
alert(li);        //결과 : li의 값 : a, b, c, d, e, f

concat(추가)

복수의 원소를 배열에 추가하는 방법입니다.

var li = ['a', 'b', 'c', 'd', 'e'];
li = li.concat(['f', 'g']);
alert(li);       //결과 : li의 값 :  a, b, c, d, e, f, g

unshift(추가)

배열의 시작점에 원소를 추가하는 방법입니다.

var li = ['a', 'b', 'c', 'd', 'e'];
li.unshift('z');
alert(li);      //결과 : li의 값 :  z, a, b, c, d, e

splice(추가 및 삭제추가)

배열의 특정구간을 추출하거나, 특정구간에 특정 배열을 추가합니다.

var li = ['a', 'c', 'd'];
li.splice(1, 0, 'b');
alert(li);      //결과 : li의 값 :  a, b, c, d
  • 1 = 'b'의 값에, 0 = 'b'라고하는 앞에 값을 넣으라는 뜻입니다.
var li = ['a', 'b', 'c'];
li.splice(1, 1, 'x' 'y');
alert(li);      //결과 : li의 값 :  a, x, y, c
  • 1개의 엘리먼트를 삭제하고 뒤에 따라오는 값을 삭제된 그위치에 순서대로 집어넣으라는 뜻입니다.

shift(제거)

배열의 첫번째 원소를 제거합니다.

var li = ['a', 'b', 'c', 'd', 'e'];
li.shift();
alert(li);      //결과 : li의 값 :  b, c, d, e

pop(제거)

배열 끝점의 원소를 제거합니다.

var li = ['a', 'b', 'c', 'd', 'e'];
li.pop();
alert(li);      //결과 : li의 값 :  a, b, c, d

sort(정렬)

순서대로 정렬하고 싶을 때 사용합니다.

var li = ['c', 'e', 'a', 'b', 'd'];
li.sort();
alert(li);      //결과 : li의 값 :   a, b, c, d, e

reverse(정렬)

역순으로 정렬하고 싶을 때 사용합니다.

var li = ['c', 'e', 'a', 'b', 'd'];
li.reverse();
alert(li);      //결과 : li의 값 :   e, d, c, b, a

객체
